<div className="errors-responses-section">
  <div className="errors-responses-card">
    <h3>Errors & Responses</h3>

    <div className="response-format">
      <h4>Response Format</h4>
      <p>All API responses are returned in JSON format with the following structure:</p>

      <div className="code-block-container">
        <pre className="code-block">

```json
{
  "status": true,
  "message": "Operation successful",
  "data": []
}
```

        </pre>
      </div>

      <div className="response-fields">
        <div className="response-field">
          <div className="response-field-name"><code>status</code></div>
          <div className="response-field-description">A boolean indicating whether the request was successful</div>
        </div>
        <div className="response-field">
          <div className="response-field-name"><code>message</code></div>
          <div className="response-field-description">A string providing information about the result of the request</div>
        </div>
        <div className="response-field">
          <div className="response-field-name"><code>data</code></div>
          <div className="response-field-description">An object or array containing the response data</div>
        </div>
      </div>
    </div>

    <div className="error-formats">
      <div className="error-format">
        <h4>Simple Error Format</h4>
        <p>When an error occurs, the API will return a JSON response with a <code>status</code> of <code>false</code> and an error message:</p>

        <div className="code-block-container">
          <pre className="code-block">

```json
{
  "status": false,
  "message": "Error message",
  "data": []
}
```

          </pre>
        </div>
      </div>

      <div className="error-format">
        <h4>Validation Error Format</h4>
        <p>When validation errors occur, the API will return a JSON response with field-specific error messages:</p>

        <div className="code-block-container">
          <pre className="code-block">

```json
{
  "status": false,
  "message": "Error message",
  "errors": {
    "field_name": ["Error message"]
  },
  "data": []
}
```

          </pre>
        </div>
      </div>
    </div>

    <p>For more details on error handling, see the <a href="/developer-guide/error-handling">Error Handling</a> guide.</p>

  </div>
</div>

<div className="pagination-section">
  <div className="pagination-card">
    <h3>Pagination</h3>

    <p>For endpoints that return lists of objects, the API uses cursor-based pagination:</p>

    <div className="code-block-container">
      <pre className="code-block">

```
GET {ApiBaseUrl()}/subscription/get?store_id={ExampleStoreId()}&limit=10&cursor=CURSOR_TOKEN
```

      </pre>
    </div>

    <div className="pagination-params">
      <div className="pagination-param">
        <div className="pagination-param-name"><code>limit</code></div>
        <div className="pagination-param-description">The number of objects to return (default: 20, max: 100)</div>
      </div>
      <div className="pagination-param">
        <div className="pagination-param-name"><code>cursor</code></div>
        <div className="pagination-param-description">A token that points to the end of the previous page</div>
      </div>
    </div>

    <p>The response will include pagination information in the <code>meta</code> field:</p>

    <div className="code-block-container">
      <pre className="code-block">

```json
{
    "status": true,
    "message": "Subscriptions retrieved successfully",
    "data": [...],
    "meta": {
        "has_more": true,
        "next_cursor": "NEXT_CURSOR_TOKEN"
    }
}
```

      </pre>
    </div>

  </div>
</div>
